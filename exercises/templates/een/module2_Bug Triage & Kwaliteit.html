Module 2 — Bug Triage & Kwaliteit van Bevindingen

In module 1 leerde je vinden. In deze module leer je scheiden: wat is belangrijk, wat is ruis, wat is reproduceerbaar, en wat is “slechts observatie”.

1. De kern van professioneel testen

Niet alles wat fout is, is een bug.
Niet elke bug is belangrijk.
Niet elke belangrijke bug is urgent.

Professioneel testen = keuzes maken. Je leert hier werken met:

  • Severity → hoe ernstig is het probleem technisch/functioneel?
  • Priority → hoe snel moet dit worden opgelost?
  • Reproduceerbaarheid → hoe betrouwbaar is het probleem?
  • Impact → wie wordt geraakt?

2. Triage-matrix

Triage-indeling

  • Blokkerend → systeem onbruikbaar
  • Kritisch → kernfunctionaliteit faalt
  • Major → ernstige fout, maar workaround mogelijk
  • Minor → cosmetisch / UX / randgeval
  • Observatie → opvallend, maar geen directe fout

3. Oefening A — Herclassificeren

Pak 3 bevindingen uit Module 1 (eigen bugs of die van anderen).

  1. Classificeer elke bevinding in één categorie:
    • Blokkerend
    • Kritisch
    • Major
    • Minor
    • Observatie
  2. Geef per item:
    • Severity (laag/middel/hoog)
    • Priority (laag/middel/hoog)

4. Oefening B — Reproduceerbaarheidstest

Kies 1 bug en test:

  1. Kun jij hem 3× achter elkaar reproduceren?
  2. Kan iemand anders hem reproduceren met jouw stappen?
  3. Kan hij hem reproduceren in een andere browser?
Als een bug niet reproduceerbaar is → is hij niet bruikbaar.

5. Oefening C — Van observatie naar hypothese

Neem één vaag probleem (“dit voelt raar”) en herschrijf het als hypothese:

Observatie:
“De pagina voelt traag.”

Hypothese:
“Bij herladen van pagina X worden 4 onnodige scripts geladen, wat leidt tot vertraagde rendering.”

6. Professioneel bug report (v2)

  • Titel: Wat + waar + effect
  • Context: Feature/flow/doel
  • Stappen: minimaal & exact
  • Verwacht: functioneel gedrag
  • Actueel: afwijking
  • Reproduceerbaarheid: altijd/soms/zeldzaam
  • Severity: laag/middel/hoog
  • Priority: laag/middel/hoog
  • Impact: gebruiker/bedrijf/systeem

7. Reflectie

  1. Welke bug leek belangrijk maar was het niet?
  2. Welke kleine bug had onverwacht hoge impact?
  3. Waar zat jouw grootste beoordelingsfout?
