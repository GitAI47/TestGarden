Module 2 — Git Basics: Repositories, Clonen & Commits

In deze module ga je leren wat een repository is, hoe je een project cloned naar je eigen laptop, en hoe je je eerste commit maakt. Dit is de praktische start van werken met Git.

1. Wat is een repository?

Een repository (ook wel “repo”) is een map waarin Git alle bestanden én alle geschiedenis van een project bewaart.

Een repo bevat onder andere:

  • alle projectbestanden
  • de volledige commitgeschiedenis
  • informatie over branches

In deze Git-Garden leeromgeving gaan we een speciale oefenrepo gebruiken:

Oefenrepository:
git-garden-playground
*(dit is de repo die jij als cursist gebruikt om alles in te oefenen)*

2. Wat is clonen?

Clonen betekent dat je een kopie van een GitHub-repo downloadt naar jouw laptop. Vanaf dat moment werk je lokaal, met je eigen tijdmachine (Git).

Clonen doe je maar één keer. Daarna werk je telkens in dezelfde lokale map.

Voorbeeld

Je clone’t de repository git-garden-playground. Op je computer verschijnt een map met dezelfde naam. Alles wat je daar verandert, wordt door Git gevolgd.

3. Wat is een commit?

Een commit is een momentopname van je project. Je maakt een commit wanneer je een kleine, logische stap hebt afgerond.

Elke commit bevat:

  • welke bestanden zijn aangepast
  • welke regels zijn toegevoegd of verwijderd
  • een beschrijving (commit message)
  • de naam van de persoon die de commit maakte

Een goede commit message begint vaak met een korte samenvatting zoals:

  • Add introduction text
  • Fix broken link
  • Update README formatting
Tip: Maak commits in kleine stapjes. Zo blijft je werk overzichtelijk, en kunnen anderen begrijpen wat je hebt gedaan.

4. Wat zijn branches?

Een branch is een zijtak van het project waarin je veilig kunt experimenteren zonder de hoofdversie kapot te maken.

De belangrijkste branch heet meestal:

  • main
  • of master

Wanneer je iets wilt aanpassen of testen, maak je meestal een nieuwe branch, zoals:

  • update-readme
  • add-testfile
  • fix-typo

Dit werkt als een zandbak waar jij kunt bouwen zonder dat iemand anders last heeft van jouw wijzigingen.

5. Praktijkopdracht: Clone en maak je eerste commit

Nu ga je zelf aan de slag in de echte oefenrepository. Dit is je eerste échte Git-actie in de Git Garden!

Opdracht

  1. Ga naar GitHub en open de repo git-garden-playground.
  2. Klik op “Code” en kopieer de clone-URL.
  3. Open je terminal en typ: 
    git clone <URL>
  4. Open de map op je computer en maak een nieuw tekstbestand:
    hello-garden.txt
  5. Voeg één regel toe:
    Dit is mijn eerste commit in de Git Garden.
  6. Sla het bestand op en maak je commit: 
    git add .
git commit -m "My first Git Garden commit"

Je hebt nu officieel je eerste “zaadje” geplant in de Git Garden.

6. Korte Quiz

Beantwoord de drie vragen om te checken of de basis blijft hangen:

  1. Wat is het doel van clonen?
    (a) Een lokale kopie maken van een GitHub-project)
  2. Wat bevat een commit?
    (b) Een snapshot van wijzigingen + een beschrijving)
  3. Wat is een branch?
    (c) Een veilige tak om wijzigingen in te testen)
