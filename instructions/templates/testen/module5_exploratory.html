Module 5 – Exploratory Testing

Exploratory Testing is één van de krachtigste en leukste manieren van testen. Je gaat op ontdekkingstocht door de software zoals een echte gebruiker dat doet: nieuwsgierig, creatief en zonder vooraf vastgelegde stappen. Dit is waar testers echte waarde toevoegen.

1. Wat is Exploratory Testing?

Exploratory Testing betekent dat je ontdekkend test: je bedenkt tegelijk wat je wilt testen, voert het uit en leert tijdens het testen.

Het is het tegenovergestelde van een strak script: je volgt je ogen, je intuïtie en de signalen van de software.

  • Je denkt als een gebruiker
  • Je klikt, probeert, experimenteert
  • Je volgt rare foutjes of onverwachte situaties
  • Je onderzoekt zonder dat er iemand zegt "doe stap 1, 2, 3"

2. Waarom is Exploratory Testing belangrijk?

Exploratory testing vindt vaak fouten die nooit in documenten of testcases staan.

Voorbeelden van problemen die je vaak alleen exploratief vindt:

  • Raar gedrag dat alleen gebeurt als je snel klikt
  • Links die op mobiel anders werken dan op desktop
  • Onlogische flows (“waar ben ik nu beland?”)
  • Stappen die intuïtief niet kloppen
  • Foutjes in tekst, opmaak of layout

Exploratory testing is dus de perfecte aanvulling op scripted testing.

3. Session-Based Test Management (SBTM)

Vaak werk je tijdens exploratief testen in sessions: korte, gefocuste blokken tijd waarin je een specifiek doel hebt.

Een session heeft drie onderdelen:

  • Charter – wat ga je onderzoeken?
  • Timer – meestal 30 tot 60 minuten
  • Note-taking – wat zie je, wat werkt niet, wat valt op?

Voorbeeld Charter

“Onderzoek de registratiepagina op onlogische flows, foutmeldingen en mobiel gebruik.”

4. Hoe doe je Exploratory Testing?

Een paar praktische handvatten:

  • Begin met een duidelijk doel (je charter)
  • Ga de software in zoals een gebruiker dat doet
  • Wees nieuwsgierig: “Wat gebeurt er als…?”
  • Volg onverwachte dingen — dat zijn kansen!
  • Noteer: foutmeldingen, gek gedrag, vertraging, vragen, ideeën
  • Test ook eens heel snel, of juist heel traag
  • Test op mobiel, tablet, andere browsers
Pro-tip: Exploratory testing is geen chaos. Je volgt een doel, maar laat ruimte voor ontdekking.

5. Session Report (korte template)

Session Report – [onderdeel] – [datum]

Charter:
  Wat was je doel?

Duur:
  Hoe lang heb je getest?

Observaties:
  - Wat zag je?
  - Welke rare situaties kwamen naar voren?

Issues gevonden:
  - Bug 1: omschrijving + hoe te reproduceren
  - Bug 2: omschrijving + hoe te reproduceren
  - Bug 3: omschrijving + hoe te reproduceren

Opmerkingen:
  Vragen, ideeën, twijfels

6. Praktijkopdracht: 60-minuten Exploratory Session

Je gaat nu zelf een volledige exploratory testing sessie uitvoeren.

Opdracht

  1. Kies een onderdeel van de demo-app (bijv. registratie, login, winkelwagen).
  2. Schrijf een charter van één zin.
  3. Zet een timer op 60 minuten.
  4. Test exploratief en noteer alles wat opvalt.
  5. Schrijf een session report.
  6. Lever minimaal 3 bugs in als losse bugreports.
Tip: Als je vastloopt, stel de vraag: “Waar zou een gebruiker in de war kunnen raken?”

7. Reflectie

Sta stil bij je sessie:

  • Wat verraste je?
  • Welke bug had je nooit gevonden met alleen testcases?
  • Welke skills kwamen goed van pas?
  • Wat wil je anders doen volgende keer?
