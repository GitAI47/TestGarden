Module 5 — Automatisering & Bewuste Keuzes

Niet alles wat testbaar is, moet geautomatiseerd worden. Niet alles wat geautomatiseerd kan worden, zou geautomatiseerd moeten worden.

1. De mythe van “alles automatiseren”

Automatisering vergroot kracht — maar ook fouten.

Slechte tests geautomatiseerd = sneller slechte informatie.

2. Wat is wél geschikt voor automatisering?

  • Stabiele flows
  • Herhaalbare tests
  • Regressietests
  • Smoke tests
  • Datavalidatie
Vuistregel:
Hoe voorspelbaarder het gedrag, hoe beter te automatiseren.

3. Wat is níet geschikt voor automatisering?

  • UX-beleving
  • Look & feel
  • Exploratory testing
  • Nieuwe features
  • Vage problemen
Mensen testen betekenis. Machines testen herhaling.

4. Oefening A — Classificeren

Neem 10 tests uit je regression set (Module 4) en label ze:

  • A = geschikt voor automatisering
  • M = menselijk testen
  • H = hybride (mens + tool)

5. Oefening B — Mini automatiseringsstrategie

Maak een mini-strategie (1 pagina):

  • Wat automatiseer je eerst?
  • Waarom juist die?
  • Wat automatiseer je bewust niet?
  • Welke risico’s heeft automatisering hier?
Voorbeeld:
“Login-flow automatiseren voor regressie, maar checkout UX handmatig testen.”

6. Tools als verlengstuk (niet als vervanging)

Tools versterken je waarneming:

  • DevTools
  • Network tab
  • Logfiles
  • Monitoring
  • Automatisering
De tester blijft altijd de denker.

7. Reflectie

  1. Waar zou automatisering jou juist blind maken?
  2. Waar zou het je juist kracht geven?
  3. Wat mag nooit volledig geautomatiseerd worden?
