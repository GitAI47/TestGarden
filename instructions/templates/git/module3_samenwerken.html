Module 3 — Pull Requests & Samenwerken

In deze module leer je hoe je via GitHub samenwerkt met anderen. Het centrale hulpmiddel daarbij is de pull request. Daarmee vraag je anderen om jouw werk te bekijken, te bespreken en uiteindelijk samen te voegen. Dit is de plek waar echte team-samenwerking begint.

1. Wat is een pull request?

Een pull request (PR) is een voorstel om jouw wijzigingen in een branch samen te voegen met de hoofdversie van het project (meestal main).

Een PR wordt gebruikt om:

  • te laten zien wat jij hebt veranderd
  • feedback te ontvangen van teamleden
  • bugs op te sporen vóórdat je merge't
  • kwaliteit en samenwerking te verbeteren

Een pull request is dus geen formaliteit — het is een gesprek.

2. Hoe maak je een pull request?

Het proces bestaat uit vier eenvoudige stappen:

  1. Maak een nieuwe branch.
  2. Doe je wijzigingen en commit ze.
  3. Push de branch naar GitHub.
  4. Klik op “New Pull Request” op GitHub.

Tijdens het openen van de PR schrijf je een korte uitleg over:

  • wat je hebt gewijzigd
  • waarom je het hebt gewijzigd
  • of er iets is dat reviewers moeten weten

Daarna kunnen anderen jouw werk bekijken, reacties achterlaten en goedkeuren.

3. Wat is een review?

Een review is het proces waarin iemand anders jouw wijzigingen bekijkt. Dit lijkt misschien spannend, maar het is juist iets positiefs: je voorkomt fouten, je leert van elkaar, en de software wordt beter.

Een reviewer let op:

  • is de code begrijpelijk?
  • heeft de wijziging onverwachte gevolgen?
  • is de bestandsstructuur logisch?
  • zijn commit messages duidelijk?

Jij kunt als tester ook uitstekende reviews geven — zeker op helderheid, logica, namen en risico’s.

Tip: Een goede pull request is klein, duidelijk en overzichtelijk. Grote PR’s zijn moeilijker te reviewen en bevatten vaker fouten.

4. Wat betekent “mergen”?

Mergen betekent dat de wijzigingen in jouw branch worden toegevoegd aan de hoofdversie (bijv. main).

Dit gebeurt pas wanneer:

  • de reviewer akkoord is
  • de PR duidelijk is uitgelegd
  • er geen conflicten meer zijn

Na het mergen kun je de branch meestal verwijderen — hij heeft zijn werk gedaan.

5. Praktijkopdracht: Maak je eerste pull request

In deze opdracht ga je een nieuwe branch maken, een kleine wijziging doen, en een pull request openen in de oefenrepo git-garden-playground.

Opdracht

  1. Open de repository git-garden-playground op GitHub.
  2. Maak een nieuwe branch, bijv. add-my-name.
  3. Maak lokaal een nieuw bestand:
    my-contribution.txt
  4. Zet in dat bestand één regel:
    Dit is mijn eerste pull request.
  5. Commit het bestand en push je branch.
  6. Ga op GitHub naar Pull Requests → New Pull Request.
  7. Schrijf een korte beschrijving:
    • Wat heb je toegevoegd?
    • Waarom heb je deze informatie toegevoegd?
  8. Dien de PR in.

Je hebt nu je eerste volledige bijdrage gedaan zoals in een echt team!

6. Korte Quiz

Beantwoord de drie vragen:

  1. Waarvoor gebruik je een pull request?
    (a) Voorstellen om jouw wijzigingen samen te voegen met de hoofdbranch)
  2. Wat doet een reviewer?
    (b) Bekijkt en beoordeelt de wijzigingen in een PR)
  3. Wanneer mag je een branch meestal verwijderen?
    (c) Zodra de PR is gemerged)
