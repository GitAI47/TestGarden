Module 1 — Wat is Git & GitHub?

Welkom in de Git Garden! Voordat we branches, commits en pull requests gaan gebruiken, is het belangrijk om te begrijpen waarom Git bestaat en waarom bijna alle softwareteams GitHub gebruiken. In deze module leggen we het fundament dat je in alle volgende stappen nodig hebt.

1. Wat is versiebeheer?

Versiebeheer betekent: bijhouden hoe bestanden veranderen in de tijd. Net zoals Google Docs alle eerdere versies onthoudt, doet Git dat ook — maar dan voor complete softwareprojecten.

Met versiebeheer kun je:

  • zien wie welke wijziging heeft gemaakt
  • teruggaan naar een eerdere versie als iets kapot is gegaan
  • veilig experimenteren zonder het project stuk te maken

Git is de meest gebruikte versiebeheer-tool ter wereld.

2. Wat is Git?

Git is een systeem dat veranderingen vastlegt in zogenaamde “commits”.
Een commit is als een foto van het project op dat moment.

Belangrijk om te weten:

  • Git werkt op jouw computer — lokaal.
  • Jij bepaalt wanneer je een foto maakt (commit).
  • Je kunt altijd terug naar een eerdere foto.

Voorbeeld

Stel je maakt een rekenmachine-app. Commit 1: bestanden aangemaakt Commit 2: optelfunctie werkt Commit 3: aftrekken toegevoegd Commit 4: bugfix Git onthoudt al deze stappen, en jij kunt ze allemaal bekijken.

3. Wat is GitHub?

GitHub is niet hetzelfde als Git — maar ze werken samen.

GitHub is een online platform waar je Git-projecten kunt opslaan zodat meerdere mensen eraan kunnen samenwerken. Het is dus een centrale plek waar het team het werk samenbrengt.

  • Je kunt er projecten delen met een team.
  • Je kunt zien wat anderen doen.
  • Je maakt pull requests om samen te werken.
  • Er zijn tools voor code reviews, issues en releases.

Git werkt op je laptop. GitHub werkt op internet. Samen vormen ze de basis van moderne softwareontwikkeling.

Tip: Denk aan Git als de tijdmachine op jouw computer en GitHub als de plaats waar alle tijdlijnen bij elkaar komen.

4. Waarom is Git belangrijk voor testers?

Testers gebruiken Git & GitHub om meerdere redenen:

  • Testscript of testdata beheren in een team
  • Documenteren van bugs en testbevindingen
  • Zelf kleine aanpassingen in projecten doen
  • Branches gebruiken om veilig testcode te maken
  • Een pull request maken voor een bugfix of testupdate

Zelfs als je niet programmeert, is Git een hulpmiddel dat je werk duidelijker, veiliger en professioneler maakt.

5. Praktijkopdracht: Bekijk een commit-historie

In deze mini-opdracht kijk je naar de commit-historie van een GitHub-project. Je hoeft nog niets te doen — alleen kijken en begrijpen.

Opdracht

  1. Ga naar GitHub.com.
  2. Open een willekeurig openbaar project (bijv. “python/cpython”).
  3. Klik bovenaan op “Commits”.
  4. Beantwoord kort voor jezelf:
    • Wat valt je op aan de commit-berichten?
    • Kun je zien wie welke wijziging maakte?
    • Zie je de chronologie van het project?

Je hoeft geen analyse te maken — alleen observeren. Zo leer je hoe echte teams Git gebruiken.

6. Korte Quiz

Beantwoord de drie vragen om te controleren of de basis helder is.

  1. Wat is het verschil tussen Git en GitHub?
    (a) Git is lokaal, GitHub is online)
  2. Wat is een commit?
    (a) Een momentopname van het project)
  3. Waarom gebruiken teams GitHub?
    (b) Omdat het samenwerken overzichtelijk maakt)
