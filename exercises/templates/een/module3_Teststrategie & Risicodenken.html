Module 3 — Teststrategie & Risicodenken

In module 1 leerde je vinden. In module 2 leerde je wegen. In deze module leer je richting geven: waar test je eerst, waarom, en hoe weet je dat je voldoende getest hebt?

1. Waarom strategie?

Je kunt nooit alles testen. Strategie helpt je kiezen: maximale risicoreductie met minimale tijd.

Teststrategie = “Waar zitten de grootste risico’s, en hoe pakken we die het slimst aan?”

2. De risicoformule

Risico = Impact × Kans

  • Impact: wat gebeurt er als dit faalt?
  • Kans: hoe waarschijnlijk is het dat dit faalt?

Je maakt nu een mini-risicomatrix voor één oefensite. Daarmee bepaal je welke onderdelen je eerst test.

3. Oefening A — Kritieke paden (Critical Paths)

Kies één oefensite (bijv. Sauce Demo of ParaBank). Schrijf 3 kritieke gebruikerspaden op:

  1. Happy flow (de belangrijkste route)
  2. Foutflow (wat als het misgaat?)
  3. Randgeval (ongewone invoer of situatie)

Voorbeeld (webshop)

  • Login → product kiezen → in winkelwagen → checkout
  • Login faalt → foutmelding → herstelpad
  • Checkout met lege velden / vreemde tekens / extreme aantallen

4. Oefening B — Risicomatrix (mini)

Maak een tabel met minimaal 6 onderdelen/features van de site. Geef per onderdeel:

  • Impact: laag/middel/hoog
  • Kans: laag/middel/hoog
  • Prioriteit: wat test je eerst?

Risicomatrix (voorbeeld)

Login (Impact: hoog, Kans: middel) → eerst testen
Zoekfunctie (Impact: middel, Kans: hoog) → vroeg testen
Footer links (Impact: laag, Kans: laag) → laat testen

5. Oefening C — Testdekking zonder “alles”

Je kiest nu één kritieke flow en maakt een kleine “coverage set”:

  1. 3 functionele tests (werkt het?)
  2. 3 negatieve tests (wat als het fout gaat?)
  3. 2 data-varianten (kort/lang, cijfers/letters, speciale tekens)
Coverage is niet “meer tests”. Coverage is “slimmer gekozen tests”.

6. Testdesign technieken (light)

Kies één invoerveld (bijv. password, bedrag, aantal) en pas 2 technieken toe:

  • Boundary Values: net onder / op / net boven een grens
  • Equivalence Partitioning: groepen input die hetzelfde gedrag geven
  • Pairwise (optioneel): combinaties zonder explode te testen

Voorbeeld (lengte veld)

Als max = 10 tekens → test 9, 10, 11.
Groepen: leeg / geldig / te lang / speciale tekens.

7. Deliverable: jouw mini testplan (1 pagina)

Lever één korte pagina in met:

  • De gekozen oefensite
  • 3 kritieke paden
  • Risicomatrix (min. 6 onderdelen)
  • Coverage set voor 1 flow (8 tests)
  • 1 testdesign voorbeeld (boundary/partitioning)
Dit is een testplan dat je in de praktijk echt kunt gebruiken.

8. Reflectie

  1. Wat testte je eerst, en waarom?
  2. Welke aanname bleek fout?
  3. Welke test gaf jou het meeste vertrouwen — en waarom?
