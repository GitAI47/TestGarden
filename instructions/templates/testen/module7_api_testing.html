Module 7 – API Testing

In deze module leer je de basis van API Testing. Veel software bestaat uit onderdelen die via API’s met elkaar praten. Als tester kun je deze API’s direct controleren met tools zoals Postman. Dat maakt je testwerk sneller, preciezer en veel krachtiger.

1. Wat is een API?

API staat voor Application Programming Interface. Het is een manier waarop verschillende onderdelen van software met elkaar communiceren.

Als je een API test, controleer je:

  • wat er gebeurt wanneer je data opvraagt (GET)
  • wat er gebeurt wanneer je data verstuurt (POST/PUT)
  • of er correcte foutmeldingen zijn
  • of de structuur van het antwoord klopt
  • of de responstijd in orde is

Voorbeeld

Een app stuurt via een API het verzoek: “Geef mij alle producten uit de categorie ‘boeken’.”
De server stuurt een lijst terug in de vorm van JSON.

2. HTTP Statuscodes

Wanneer je een API-aanroep doet, krijg je altijd een statuscode terug. Deze vertelt of de aanvraag gelukt is of niet.

  • 200 – OK (alles ging goed)
  • 201 – Created (nieuwe data aangemaakt)
  • 400 – Bad Request (de client doet iets verkeerd)
  • 401 – Unauthorized (geen toegang)
  • 404 – Not Found (bestemming bestaat niet)
  • 500 – Server Error (fout in de backend)
Tip: Als je een 500 krijgt, is dat bijna altijd een serverfout — en dus een belangrijke bug.

3. JSON: De taal van API’s

De meeste API’s sturen data terug in JSON formaat. Dat ziet eruit als gestructureerde tekst. 

{
  "id": 42,
  "name": "Testproduct",
  "price": 9.99
}

Tijdens het testen controleer je:

  • staan alle velden erin?
  • kloppen de waardes?
  • zit er onverwachte data in?

4. API’s testen met Postman

Postman is een gratis tool waarmee je eenvoudig API-aanvragen kunt doen.

Je kunt bijvoorbeeld testen:

  • of een endpoint bestaat
  • of een API correct reageert op foutieve input
  • hoe snel de server reageert
  • of JSON-schema’s kloppen

Basisaanvraag in Postman

  1. Open Postman
  2. Kies: GET
  3. Vul URL in: https://example.com/api/products
  4. Klik op “Send”
  5. Bekijk statuscode, headers en body

5. Basis API-checklist

Gebruik deze checklist bij het testen van API’s:

  • [ ] Bestaat het endpoint (geen 404)?
  • [ ] Reageert het correct op goede input (200/201)?
  • [ ] Reageert het correct op foute input (400)?
  • [ ] Is de structuur van de JSON correct?
  • [ ] Kloppen de waardes inhoudelijk?
  • [ ] Reageert het snel genoeg?

6. Praktijkopdracht: 6 API-checks

Je gaat nu zelf zes API-tests uitvoeren met Postman.

Opdracht

  1. Kies een openbaar API-endpoint of de demo-API van de docent.
  2. Voer drie positieve tests uit (bijvoorbeeld geldige GET-aanvragen).
  3. Voer drie negatieve tests uit (bijvoorbeeld foutieve input).
  4. Noteer:
    • statuscode
    • response time
    • response body
  5. Beschrijf eventuele afwijkingen of bugs.
Tip: Foutcodes zijn niet “fouten” — ze zijn vaak juist een teken dat de API netjes reageert.

7. Reflectie

  • Welke statuscodes zag je het meest?
  • Was de API voorspelbaar of verrassend?
  • Welke negatieve test leverde de interessantste reactie op?
  • Waar zou jij een beveiligingsrisico vermoeden?
